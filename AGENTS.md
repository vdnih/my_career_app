# AI Agent Work Guidelines

## 1. Documentation First
`docs/` ディレクトリ内のドキュメント（アーキテクチャ方針、テスト方針など）を十分に理解した上で、作業を開始すること。

## 2. Test-Driven Development (TDD)
原則としてテスト駆動開発（TDD）を行うこと。
テストコードは単なる動作検証だけでなく、プロジェクトの「機能一覧」および「技術ドキュメント」を兼ねるように設計・実装すること。

## 3. Self-Maintenance of Guidelines
ユーザーの指示内容が、単発のタスクではなく「今後の継続的な対応」や「プロジェクト全体の基本方針」に関わるものであると判断した場合、あるいはユーザーから明確に「ルールに追加して」と指示された場合、エージェントは自律的にこの `AGENTS.md` ファイルを編集し、適切なセクションに該当ルールを追記したパッチ（差分）を提案すること。
